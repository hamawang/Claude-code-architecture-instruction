# Claude Code 终端端到端 (End-to-End) 工作流总览

当您在终端中敲下一句指令（比如：_"帮我查一下项目里 `QueryEngine` 的问题并写个分析说明书"_）并按下回车，这背后发生了一场由无数精密齿轮咬合的交响乐。

结合我们刚才编写的全部架构分析文档，以下是 Claude Code 从接收消息到最后落盘记忆的**完整生命周期七大阶段**：

```text
+---------------------------------------------------------+
| [阶段 1] 终端接收与命令分发                             |
| entrypoints/cli.tsx & /commands                         |
| (斜杠命令本地直接拦截执行 / 普通对话向下流转)           |
+---------------------------------------------------------+
                            |
                            v
+---------------------------------------------------------+
| [阶段 2] 极速物理环境插桩                               |
| context.ts & QueryEngine.ts                             |
| (挂载本地时间戳、git 状态快照、项目核心 CLAUDE.md)      |
+---------------------------------------------------------+
                            |
                            v
+---------------------------------------------------------+
| [阶段 3] 提示词预检与缓存折叠                           |
| microCompact.ts / autoCompact.ts                        |
| (清理过期工具冗余 -> 容量濒危时强行偷换为后台概括摘要)  |
+---------------------------------------------------------+
                            ||
                            vv
+---------------------------------------------------------+
| [阶段 4] 大模型推理主循环开启                           |
| query.ts / queryLoop                                    |
| (流式渲染思考文字给前台 -> 决定调用底座工具 Tool Use)   |
+---------------------------------------------------------+
               |                           |

+---------------------------+ +---------------------------+
| [阶段 5] 工具审查与沙盒阵 | | [阶段 6] 多轮缠斗与自愈环 |

| (执行防线 / 拦截报错自愈)   | | (携新线索再查询 / 宣布停战) |

+---------------------------------------------------------+
```

---

## 阶段 1：接收、拦截与宏观拦截器 (`entrypoints/cli.tsx` & `/commands`)

1. **终端捕获**：终端界面 (通常由 React + Ink 构建) 捕获到您的按键输入文本。
2. **斜杠命令路由 (Slash Commands)**：系统先做一把正则探测 —— 用户输入的是不是以 `/` 开头？
   - 如果是 `/compact`, `/clear`, `/bug`, `/logout` 等，系统直接拦截，调用本地同步函数执行对应的动作。大模型甚至不知道你敲过这些字。
   - 如果是普通文本，进入主引擎。

## 阶段 2：环境感知与物理世界挂载 (`context.ts` & `QueryEngine.ts`)

1. **实例化唤醒 (`QueryEngine` )**：会话配置总调度室接手任务，给这次请求贴上防篡改的 UUID 并计入账单表。
2. **物理世界贴图 (`getUserContext` / `getSystemContext`)**：
   在把你的那句“帮我查一下代码...”发给大模型之前，系统偷偷地做了手脚，在你的提示词头部塞进了一坨隐藏信息：
   - 包含你当前电脑的本地时间戳（防幻觉）。
   - 自动运行并抓取了当前目录的 `git status`、`git log` 状态。
   - 抓取了当前项目根目录下的知识库 `CLAUDE.md`。

## 阶段 3：缓存审计与阈值报警器 (`autoCompact.ts`)

在按下发送按钮的最后一刻，防爆探头开启扫描历史对话池子的总 Token 数：
1. **微核删减 (`microCompact.ts`)**：如果时间间隔过长，偷偷删掉上一次超长的 Bash/ls 执行结果（[Old tool result content cleared]）以节省发车油费。
2. **重度重组 (`sessionMemoryCompact` / `compact`)**：如果字数突破了阈值（比如高达原本 90% 容量），主线程紧急挂起 —— 抽出后台早就写好的 `SESSION_MEMORY.md` 摘要作替换，甚至抛弃前面大部分上下文，强制折叠前缀后，再发车。

## 阶段 4：大模型推理主循环开启 (`query.ts` : `queryLoop`)

进入到流式推断机的心智阶段。这里的 `async function*` 开始大显神威：
1. 大模型开始输出**文字 (text)** 和 **工具块 (tool_use)**。
2. 流式返回的文本被立刻送到前面 UI 线程渲染打印，让用户感觉“它马上就有反应了”。
3. 当模型觉得必须查看代码才能回答你时，它抛出了一个 `tool_use` (意图使用某个工具)。

## 阶段 5：插件生态底座与拦截审批 (`Tool.ts` : Permissions)

大模型决定的 `tool_use` 会撞击安全墙。
1. **权限防火墙判断**：如果大模型想跑 `BashTool` (比如 `npm install`)，系统调用它的 `alwaysAllow` / `alwaysAsk` 规则验证。遇到破坏性修改操作，系统自动中断流并弹窗：“*AI 刚才申请执行 `npm i`，阁下是否批准 (y/N)？*”。
2. **沙盒执行 (`Tool.call()`)**：如果用户授权，或是安全动作（比如只读的 `grep_search`），该工具立即在电脑本地执行。
3. **失败闭合 (Fail-Closed/自愈)**：如果命令输错了（比如用错参数），工具捕获报错文本传给 Agent，依靠强大的 `catch` 环境让它“看清错误代码再试一次”。

## 阶段 6：多轮缠斗与主动完结 (Multurn & Recursion)

1. **Tool Result 返回**：执行反馈给到模型，`query.ts` 携带新结果发起下一个 API 请求（进入下一个 while 循环）。
2. **递归思考**：AI 继续审查自己的查寻结果，如果不够，则再次下发新的 `ToolUse`。（这就是你看它思考转半天、不断查文件的本质）。
3. **达成终点 (Stop Reason)**：当模型分析完你的所有源码后，它输出最终的答复文字，且没有带出任何工具调用动作，API 返回 `stop_reason: "end_turn"`。系统宣布该回合胜利停转。

## 阶段 7：幽灵善后与潜意识凝练 (`extractMemories` & `sessionMemory`)

这一步发生在你已经收到回复、主大屏静止的时候（极其丝滑，不会影响你）。两个后台系统通过**不同的钩子管线**分别触发：
1. **临时会话记录 (`sessionMemory.ts`)**：通过 `registerPostSamplingHook` (Post Sampling Hook) 挂载触发。派发出一个被严管了写权限的分身子代理 (Forked Agent)，在后台偷窥着刚才所有的聊天走势，快速编辑位于 `~/.claude/` 体系下的会话记忆文件供下次折叠上下文用。
2. **长期记忆凝练 (`extractMemories.ts`)**：通过 `handleStopHooks`（位于 `stopHooks.ts`）触发，与 sessionMemory 走的是不同的钩子管线。如果里面发现你刚才骂了它，或者制定了新的项目开发规则，第二个跨期长时记忆体上线，在不消费重新阅读 Token 钱的情况下（Prompt Cache Hit），静默生成卡片索引存入 `~/.claude/projects/xxx/memory/` 。将来哪怕重启 CLI 这个幽灵仍然记得。

---

这就是一个指令抛出后的整个生命旅程：**从外层的宏观命令分发，走入严苟的环境过滤与空间提纯，再坠入核心与底层的多次读写缠斗角力，最后在一场悄无声息的断点存档回写中落幕。**
