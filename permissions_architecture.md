# 权限路由引擎、YOLO 模式与 yoloClassifier 全貌分析

## 1. 核心概述
在 Agentic Code 环境中，“权限控制与安全”是最关键的命题之一。Claude Code 实现了一套复杂的权限路由引擎，用于研判工具调用和命令执行的安全性，并根据用户的授权策略采取放行、拦截或确认。其中，**YOLO 模式 (You Only Look Once)** 以及对应的 **yoloClassifier (智能提权分类器)** 是核心创新点。

## 2. 权限路由引擎与基础权限验证
权限引擎主要位于 `src/utils/permissions/` 目录下，扮演门神角色：
- **规则解析与加载：** `permissionsLoader.ts` 负责从持久化存储（如本地权限记录文件）读取规则。
- **匹配与过滤：** `permissionRuleParser.ts` 将具体的终端命令或特定请求内容提取为特征，并和现有白名单或黑名单进行正则匹配比对。
- **阻断开关机制：** `bypassPermissionsKillswitch.ts` 负责一些内部调试或特殊条件下的权限越过处理。

## 3. YOLO 模式的工作原理
YOLO 模式允许 Claude Code 减少繁琐的人工确认（Auto-Approve），从而实现完全自主的循环执行（尤其是执行非破坏性的高风险探测或是频繁迭代的编译运行任务）。
- **开启机制**：用户通过命令行标志（`-y, --yolo`）或者特殊对话指令设定。这会修改运行时的全局状态。
- **风险隔离**：虽然处于 YOLO 模式，仍可能对 rm -rf 等毁灭性操作进行系统层面的白名单校验或是更深度的模型特征校验。

## 4. yoloClassifier 系统全貌
YoloClassifier 的定位是对即将执行的意图进行语义分类和风险打分。
- 分类器本身很可能是基于规则的静态扫描（Static Linting）与轻量化模型的风险推断相结合。
- 其作用是在 YOLO 模式被触发时，对“绝对高危但尚未在人类拒绝名单中的操作”进行紧急刹车，充当安全网。
